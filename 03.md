# CORD-03: Channels

A Channel is one message plane inside a Community, *defined* in the Control Plane but carrying its own chat on an independent Chat Plane (CORD-02), a Community is simply a set of these unified under a common Control Plane.

Channels come in two kinds, differing only in *who* may read them:

**Public**: readable by every member, its key is *derived* from the CommunityRoot, so nothing is ever delivered and it rotates for free whenever the base does.

**Private**: readable only by granted role-holders, its key is an *independent* random secret, delivered on grant and rekeyed on removal (CORD-06).

----

## 1. Keying

Every Channel is addressed and encrypted the same way, through the one `group_key` derivation (CORD-02), only its *secret* and *epoch* differ by kind:
```
Public  channel_pk = group_key("concord/channel", CommunityRoot, channel_id, root_epoch)
Private channel_pk = group_key("concord/channel", channel_key,   channel_id, channel_epoch)
```
Because `channel_id` is folded into the derivation, every Channel gets its own distinct address regardless of which secret feeds it, so a Public Channel is really just "a Channel whose key derives from the CommunityRoot" rather than a separate mechanism.

A Public Channel needs no access key generation nor distribution, it's inherently readable by every member by definition, so deriving from the CommunityRoot means it needs no delivery, adds nothing to an invite, and a single Refounding (CORD-06) severs it along with the base, no per-channel rekey. A Private Channel needs real independence (a leaked channel key exposes only that one Channel, and it can be rotated independently), so it carries its own key and its own epoch.

## 2. Metadata

A Channel is *defined* by a **ChannelMetadata** entity in the Control Plane (CORD-04), holding its `channel_id`, name, and `private` flag, and edits fold by the same versioned editions as any control state, so a rename or a visibility change is an authorized, convergent edit rather than a new Channel.

Converting a Public Channel to a Private Channel is a key rotation: the Channel mints its own independent key, delivered to the desired role-holders, and its metadata flag flips, both bind as a single authorized action (`MANAGE_CHANNELS`).

Converting from Private to Public reverses it, the Channel simply begins deriving from the CommunityRoot going forward, and a member joining after the switch reads only the now-public history, never the prior private messages (since they never held that key).

## 3. Messages

A Channel's Chat Plane carries the ordinary append events (messages, reactions, edits, deletes), each wrapped exactly as a Stream event (CORD-01), signed by the Channel's `group_key` and sealed to the author's real identity within:
```json
{ "kinds": [1059], "authors": ["<channel_pk per held epoch>"] }
```
Clients load a Channel newest-first and paginate backwards, querying every epoch pubkey they hold (a Public Channel across the base epochs, a Private one across its own), so history spanning a rekey stays continuous.
